# 饮食规划 Web 端 - 流程优化方案

## 当前问题分析

### 问题 1: 字体颜色
**现象:** 暗黑模式下看不清
**原因:** Tailwind 默认是亮色，暗黑模式下没适配
**方案:**
- 添加 `dark:` class 支持
- 或者让用户手动切换主题
- 简化处理: 统一用深色文字

### 问题 2: 档案流程
**现象:** 档案是长期信息，每次都要填太麻烦
**方案:**
- 首次: 填写档案 → 保存到 localStorage
- 后续: 直接显示"使用上次档案"或"修改档案"
- 新流程: 选择档案 → 填写计划 → 生成

### 问题 3: 忌口/不爱吃
**现象:** 
- 自定义禁忌有时不生效
- 不爱吃应该跟忌口放一起
**方案:**
- 统一放档案页
- 分为两层:
  - 忌口 (绝对不能吃)
  - 不爱吃 (尽量避免)

### 问题 4: 计划页细节
**现象:** 餐次选择不够细
**方案:**
- 早餐/午餐/晚餐 增加典型示例
- 可以选"随便"或自定义

## 优化后的流程

```
┌─────────────────────────────────────┐
│  1. 欢迎页                          │
│  [使用上次档案] [新建档案]           │
├─────────────────────────────────────┤
│  2. 档案页 (长期)                   │
│  - 目标 (单选)                      │
│  - 忌口 (多选+自定义)              │
│  - 不爱吃 (多选+自定义+程度)        │
│  - 厨具 (多选)                     │
│  [保存档案]                         │
├─────────────────────────────────────┤
│  3. 计划页 (本次)                   │
│  - 风格 (多选)                      │
│  - 天数/人数/餐数/预算              │
├─────────────────────────────────────┤
│  4. 生成 → 结果展示                  │
└─────────────────────────────────────┘
```
