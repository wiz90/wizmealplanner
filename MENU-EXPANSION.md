# 饮食规划 Web App - 菜单扩展 PRD

## 目标
将菜单从"单次生成"升级为"可持续管理的饮食档案"

---

## Phase 1: P0 核心功能 (Week 1-2)

### 1.1 周菜单视图

**功能描述**
- 日历/周视图展示每天早午晚三餐
- 可视化一周饮食结构

**UI 设计**
```
周一        周二        周三        ...
[早餐]      [早餐]      [早餐]
[午餐]      [午餐]      [午餐]  
[晚餐]      [晚餐]      [晚餐]
```

**交互**
- 点击某餐可以查看详情/替换
- 支持横向滑动查看多周

---

### 1.2 历史收藏夹

**功能描述**
- 对生成的食谱一键收藏
- 支持分类：常做/快手/待尝试
- 支持打标签："无辣"、"高蛋白"

**存储**
- 使用 localStorage 持久化
- 结构:
```json
{
  "favorites": [
    {
      "id": "uuid",
      "recipe": {...},
      "tags": ["快手", "无辣"],
      "category": "常做",
      "createdAt": "2026-03-25"
    }
  ]
}
```

---

### 1.3 一键生成整周

**功能描述**
- 输入周目标（省钱/高蛋白/川味周）
- AI 一次性输出 7 天 × 3-5 餐的完整规划

**实现**
- 复用现有 generate API
- 增加 `week_mode` 参数
- 返回完整的周计划

---

## Phase 2: P1 差异化功能 (Week 3-4)

### 2.1 备菜清单智能合并

**功能描述**
- 基于整周菜单自动合并同类食材
- 生成采购清单

**当前状态**
- 已实现基础版本
- 需优化：同名不同量 的合并逻辑

---

### 2.2 单菜替换

**功能描述**
- 周菜单中某道不想吃
- 点"换一道"重新生成

---

### 2.3 营养概览

**功能描述**
- 显示本周菜单的营养估算
- 蛋白质/碳水/脂肪/热量

---

## Phase 3: P2 留存功能 (Week 5+)

### 3.1 家庭成员口味档案
### 3.2 季节/节日主题
### 3.3 剩余食材推荐
### 3.4 用餐记录

## 技术实现

### 数据结构
```typescript
interface WeekPlan {
  id: string;
  startDate: string; // 周一日期
  days: DayPlan[];
}

interface DayPlan {
  dayIndex: number;
  meals: MealPlan[];
}

interface MealPlan {
  type: '早餐' | '上午茶' | '午餐' | '下午茶' | '晚餐';
  dishes: Dish[];
}
```

### 存储策略
- **WeekPlan**: localStorage (最多保留 4 周)
- **Favorites**: localStorage (无上限)
- **UserPreferences**: localStorage (现有)
